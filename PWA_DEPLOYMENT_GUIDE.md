# خطة نشر تطبيق Maestro كـ PWA

## نظرة عامة
تم تحويل تطبيق Maestro إلى Progressive Web App (PWA) يدعم العمل في وضع عدم الاتصال بالإنترنت مع مزامنة البيانات تلقائياً عند عودة الاتصال.

## الميزات المضافة

### 1. دعم PWA
- **Service Worker**: للتخزين المؤقت والعمل offline
- **Web App Manifest**: لتثبيت التطبيق على الأجهزة
- **Background Sync**: مزامنة البيانات في الخلفية

### 2. التخزين المحلي المتقدم
- **IndexedDB**: تخزين البيانات المعقدة محلياً
- **Offline Queue**: حفظ العمليات المعلقة للمزامنة لاحقاً
- **Data Caching**: تخزين البيانات المؤقتة للوصول السريع

### 3. مزامنة البيانات
- **Automatic Sync**: مزامنة تلقائية عند عودة الاتصال
- **Conflict Resolution**: حل تعارض البيانات
- **Background Sync**: مزامنة في الخلفية حتى لو كان التطبيق مغلق

## متطلبات النشر

### 1. خادم ويب يدعم HTTPS
- Netlify
- Vercel
- Firebase Hosting
- GitHub Pages (مع HTTPS)
- خادم خاص مع شهادة SSL

### 2. إعداد الأيقونات
قم بإنشاء أيقونات بأحجام مختلفة ووضعها في مجلد `public/icons/`:


### 3. إعداد API Backend (اختياري)
إذا كنت تريد مزامنة البيانات مع خادم مركزي:

```javascript
// أضف هذه المتغيرات في .env
VITE_API_BASE_URL=https://your-api-domain.com/api
VITE_ENABLE_SYNC=true
```

## خطوات النشر

### 1. بناء التطبيق
```bash
npm run build
```

### 2. نشر الملفات
قم بنشر محتويات مجلد `dist/` على خادم الويب الخاص بك.

### 3. إعداد HTTPS
تأكد من أن موقعك يستخدم HTTPS (مطلوب لـ PWA).

### 4. اختبار PWA
- افتح الموقع في المتصفح
- افتح Developer Tools → Application → Service Workers
- تأكد من تسجيل Service Worker
- جرب تثبيت التطبيق من قائمة المتصفح

## كيفية العمل Offline

### 1. التخزين المحلي
- جميع البيانات المحلية محفوظة في IndexedDB
- التطبيق يعمل بالكامل بدون اتصال بالإنترنت

### 2. المزامنة
- عند إجراء تغييرات في وضع offline، يتم حفظها محلياً
- عند عودة الاتصال، يتم مزامنة البيانات تلقائياً
- في حالة فشل المزامنة، يتم إعادة المحاولة في الخلفية

### 3. مؤشرات الحالة
- أيقونة WiFi في الهيدر تظهر حالة الاتصال
- رسائل إشعار عند بدء/انتهاء المزامنة

```json
// في public/manifest.json
{
  "name": "اسم التطبيق المخصص",
  "short_name": "الاسم المختصر",
  "theme_color": "#your-color"

```

## استكشاف الأخطاء

### 1. Service Worker لا يعمل
- تأكد من HTTPS
- تحقق من Developer Tools → Application → Service Workers
- تأكد من وجود ملف sw.js في الجذر

### 2. التطبيق لا يعمل offline
- تأكد من تفعيل Service Worker
- تحقق من Cache Storage في Developer Tools
- تأكد من وجود manifest.json

### 3. مشاكل المزامنة
- تحقق من اتصال الإنترنت
- راجع Console للأخطاء
- تأكد من صحة API endpoints
